## 组件说明

### 控制平面（Controller）

控制平面是系统的中心枢纽，负责：

- **配置管理**: 从 Git 仓库加载服务定义
- **状态聚合**: 收集所有代理的状态信息
- **任务调度**: 将部署任务分配给适当的代理
- **API 服务**: 提供 Web UI 和外部集成接口

### 执行代理（Agent）

代理运行在目标 Docker 主机上，负责：

- **心跳通信**: 定期向控制平面报告状态
- **任务执行**: 执行部署、停止、重启等操作
- **日志收集**: 收集和转发容器日志
- **资源监控**: 监控主机和容器资源使用情况

### 通信协议

Composia 使用 ConnectRPC 进行服务间通信，提供：

- 基于 HTTP/2 的双向流
- Protobuf 序列化
- 兼容 gRPC 和 REST 客户端

## 数据流

### 部署流程

1. 用户通过 Web UI 或 API 发起部署请求
2. 控制平面验证服务定义
3. 控制平面选择目标代理节点
4. 任务进入队列等待执行
5. 代理获取任务并执行部署
6. 代理报告执行结果

### 状态同步

- 代理每 5 秒发送一次心跳
- 心跳包含节点状态和容器列表
- 控制平面聚合所有代理的状态
- Web UI 实时显示最新状态

## 安全性

- 代理使用 Token 认证
- 所有通信使用 TLS 加密（生产环境）
- 最小权限原则：代理只能访问被分配的服务
